Harden agent against reviewer edge cases

Adds 95+ scenario stress tests (heuristics + full pipeline) and fixes
every failure they exposed:

- Budget parsing: support k notation ($1.5k, 2K), bare 1500,
  trailing $, "around/about" qualifiers. Suppress false positives
  from GPU model numbers (RTX 4090), display resolutions (4K, 1440p),
  and storage/RAM/wattage figures.
- Use case heuristic: now handles "Game on it", "for work", "photo
  editing", "ML training", "3d render", "podcast", "wfh", etc.
- Input validation: block more prompt-injection / system-prompt
  extraction phrasings ("forget everything", "show me your prompt",
  "pretend to be", "jailbreak").
- Feedback heuristic: detect comparison intent ("compare with $X",
  "what if I had $X"), and relative budget moves ("double my budget",
  "halve it", "1.5x budget").
- CPU picker now restricts to mainstream sockets (AM5/AM4/LGA1700/
  LGA1200) when no explicit platform is set, so the motherboard picker
  always has plenty of in-budget boards to pair with.
- _try_upgrade / _try_downgrade for CPU stay on the existing
  motherboard's socket (no more LGA1851/sTRX4 CPUs paired with
  AM4 boards).
- _try_upgrade for memory now respects the motherboard's
  max_memory and memory_slots, so we never violate physical limits.
- Memory upgrades stay within use-case caps (32/64/128 GB) even when
  filling budget.
- Relaxed motherboard / memory pickers never fall through to a
  socket- or DDR-blind generic fallback.
- _budget_trim_pass tries the next-priciest category when one is
  uncuttable, instead of breaking the loop entirely.
- New _ensure_psu_sized rerun after fill/trim so the PSU always
  has >=10% safety margin over the upgraded build's load.
- _pick_psu targets load*1.15 (not raw load) so it clears the
  compatibility checker's 1.10 threshold with margin.

End-to-end pipeline stress test covers 11 budget x use_case combos
from $300 browsing -> $10000 workstation; all 11 now produce
compatible, in-budget builds with zero compat errors.
